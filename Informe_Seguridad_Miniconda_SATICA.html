1. ¿Qué es Miniconda y por qué lo usa SATICA?

1.1 Definición

Miniconda es un gestor de entornos de Python de código abierto, distribuido por Anaconda, Inc. Es una versión mínima de Anaconda que permite instalar librerías de Python de forma aislada y reproducible. No es malware, no es spyware, no recopila datos personales.

1.2 Función Específica en SATICA V2.0

Miniconda cumple una única función en SATICA: servir como puente para que R pueda comunicarse con la API de Google Earth Engine (GEE) a través de la librería Python earthengine-api (ee). Esta comunicación la orquesta el paquete R llamado rgee usando reticulate.

RStudio (R) → reticulate → Miniconda (Python) → Google Earth Engine API

Miniconda actúa solo como intermediario técnico entre R y Google Earth Engine.

1.3 ¿Qué hace exactamente en SATICA?

El archivo R/api_sentinel_rgee.R usa Miniconda para:

  1. Descargar imágenes satelitales Sentinel-2 (ESA/Copernicus) procesadas por Google Earth Engine.
  2. Calcular índices de vegetación (NDVI y NBR) para evaluar el estado de resequedad de los cultivos de caña.
  3. Exportar estadísticas zonales (promedio de verdor por suerte de caña) a un archivo CSV local.
✅ Miniconda NO:
  • Envía información personal, institucional o confidencial de la CVC.
  • Accede a redes internas, bases de datos corporativas ni carpetas compartidas.
  • Instala servicios en segundo plano, demonios ni puertos de escucha.
  • Modifica el registro de Windows ni la configuración del sistema operativo.
  • Recopila telemetría, analytics ni métricas de uso.

8. Conclusión y Certificación

✅ CERTIFICACIÓN DE SEGURIDAD:

El sistema SATICA V2.0, incluyendo el componente Miniconda, no representa un riesgo de seguridad informática para la infraestructura de la CVC por las siguientes razones:

  1. No envía datos confidenciales — Solo transmite coordenadas geográficas públicas.
  2. No abre puertos de escucha — Toda comunicación es HTTPS saliente (443).
  3. No instala servicios en segundo plano — No hay demonios, tareas programadas ni agentes persistentes.
  4. No accede a recursos corporativos — No toca Active Directory, correo, bases de datos ni carpetas de red.
  5. Todo el software es de código abierto — Auditable públicamente por cualquier persona.
  6. Miniconda es opcional — El 95% de SATICA funciona sin él.
  7. Los destinos son instituciones científicas — NASA (gobierno de EE.UU.) y Google Earth Engine (plataforma científica).
